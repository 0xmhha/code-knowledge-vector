1. **bgeonnx 실측** (`evaluation-design §6.4` Step 1-5)
   - 산출: r@5/MRR delta 측정값 + ADR-006 status 결정 (Accepted-default-off / Proposed 유지)

2. **NEW-2** (병행 가능, 측정 cold-start / 빌드 시간 동안 코딩)
   - 산출: `ckv eval --record` 동작 + fixture growth 패턴
   - 시간: 2-3시간

> **이유**: bgeonnx 다운로드/측정이 외부 의존 (네트워크, CoreML compile 변동) 으로 불확실.

3. **bgeonnx 실측** (시간 여유 시)
4. **Wave D / NEW-3** (별도 세션)

1. **NEW-3** PR corpus indexing (~400 LOC) — schema 변경 + `ChunkPRBackground` / `ChunkPRSolution` / `ChunkCommitMessage` 신규 ChunkKind
2. **NEW-6** Symbol-level PR breadcrumb (~80 LOC) — `Chunk.RecentPRs []PRRef` 필드 + DB column 추가
3. **NEW-7** `cks.context.related_changes` MCP tool (~150 LOC) — read-only handler, NEW-3+6 데이터 위에

**위험**: Wave A/B 안정성 + ADR-006 supersede 결정 후 진입 권장.

## 4. 세션 운영 전략

### 같은 세션에 묶을 수 있는 작업

- **bgeonnx 실측 + NEW-2** — 측정의 cold-start / 빌드 시간 동안 NEW-2 코딩. 서로 영향 없음.
- **NEW-3 + NEW-6** — schema 변경이 연속적. 한 세션에 묶어 ADR + commit 일관성 유지.
- **A2 + A4** — 둘 다 모델 도입 영역, `ckv model fetch` 로 A4 의 모델 다운로드 자동화 가능.

### 분리 세션 권장 작업

- **NEW-3** ← schema 변경 영향 큼, 모든 ChunkKind switch site 변경 + DB migration. 회귀 격리 필요.
- **B10** (parser fuzz) ← 5 파서 × 40 LOC + CI workflow 설계. 인프라 작업으로 독립.
- **B7** (Symbol ID 정규화) ← CKG repo 와의 협업, CKV 단독 결정 불가.

### 연쇄 의존 — 한 세션 또는 분리 세션

```
NEW-3 (PR corpus)
   ↓ (chunk store schema 위)
NEW-6 (PR breadcrumb)
   ↓ (검색 가능 데이터)
NEW-7 (MCP tool)
```

`NEW-3 → NEW-6` 은 한 세션 또는 두 세션. `NEW-7` 은 NEW-3+6 안정화 후 별도 세션 권장 (handler 만 추가하므로 작음).

## 5. 결정 포인트 — open questions

다음 결정이 후속 작업의 path 를 가른다.

### 5.1 bgeonnx 모델 다운로드 시점

- 본 세션: **보류** (사용자 선택, 1.34GB 다운로드 회피)
- 다음 진입: `evaluation-design §6.4 Step 1` 부터 실행 가능

**근거**: 측정 세션 전용 확보 의도. 5GB RAM + CoreML cold-start 1-3분 + N=50 측정 latency 누적 10-20분.

### 5.2 NEW-3 `schema_version` bump 정책

| 옵션 | 의미 | trade-off |
|---|---|---|
| **A. Additive** (필드 추가만, 1.x bump) | 기존 chunks DB 호환. PR 관련 신규 ChunkKind 만 *추가* 인덱싱. | 기존 ChunkKind switch 사이트는 default case 로 폴백 — 일관성 부담. |
| **B. Major bump** (2.x) | 기존 모든 chunks 무효화, `ckv reindex` 강제. PR + 기존 chunk schema 완전 일관. | 다운타임 + 사용자 reindex 부담. |

**권장 검토 시점**: Wave D / NEW-3 진입 직전. ADR 신설 (ADR-007 SCHEMA_VERSION 정책) 권장.

### 5.3 CKG repo 협업 채널 / 시점

- B7 (Symbol ID 정규화) 가 막힘 — CKV 단독 결정 불가
- pending-work §3.1 의 CKG ↔ CKV integration evaluation 도 같은 의존
- 협업 채널: PR / issue / 직접 동기화?

**권장**: B7 진입 *전* CKG repo 측에 spec proposal PR 또는 issue 로 신호. NEW-3 ChunkPRBackground 와도 의미 정렬 가능.

### 5.4 ANE 친화 모델 도입

- `pending-work-2026-05-21.md §1 #7` 의 차단: bgeonnx CPU 0.74 c/s → throughput buffer 부족
- 이전 세션의 HF 차단은 현 환경에선 풀린 것으로 보임 (HF reachable)
- EmbeddingGemma-300M 등 ANE 친화 모델 도입 재시도 가치

**권장**: bgeonnx baseline 측정 (Path A Step 1) 후 throughput 실측 → 5 c/s 미만 시 모델 교체 검토.

### 5.5 본 5 commits push 시점

origin/main 보다 5 commits ahead. 사용자가 명시적으로 `git push` 또는 PR 생성 요청 시까지 로컬 유지.
